# Junte-se ao Movimento

> **Devs cansados de vendor lock-in estao construindo algo diferente.**

---

## O Problema Coletivo

Todo dev que trabalha com LLMs já passou por isso:

- Reescrever código porque a API mudou
- Ficar preso a um provedor caro
- Perder horas adaptando Tool Calling
- Nao ter plano B quando o servico cai

**Nao é só voce. É todo mundo.**

E as big techs nao vao resolver isso — elas lucram com o lock-in.

---

## A Solucao Open Source

**ForgeLLMClient** é um SDK open source que coloca o controle de volta nas maos dos devs.

- **100% OSS** — MIT License, sem truques
- **Comunidade-first** — Issues, PRs, plugins bem-vindos
- **Zero mágica** — Código explícito, sem surpresas
- **Documentacao real** — Exemplos que funcionam

---

## Quem Está Construindo

O ForgeLLMClient nasceu da dor real de devs que:

- Ja reescreveram código demais por causa de APIs instáveis
- Querem flexibilidade sem adotar frameworks monstruosos
- Acreditam que LLMs deveriam ser intercambiaveis

**Nao é um projeto de big tech. É um projeto de devs para devs.**
